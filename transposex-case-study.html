Transpose X Case Study

Introduction

Proof-of-Concept Mobile App with Optical Music Recognition (OMR) Integration

Transpose X is a prototype application I designed to help musicians digitize and manipulate sheet music. The project combined iOS mobile development with a Node.js backend integrated with the Audiveris OMR engine to convert images into editable MusicXML files.

Problem

Musicians often need to digitize sheet music for practice, editing, or transposition. Manual transcription is time-consuming and error-prone, with no widely accessible tools available on mobile.

Solution

Transpose X processes an uploaded photo of sheet music, runs it through an OMR backend, and outputs MusicXML that can be edited or transposed in any desired key.

Technologies

  • Swift (iOS app)
  • Node.js + Express backend
  • Audiveris OMR (Java-based)
  • Dockerized backend deployment

Challenges

  • Resolving Java version incompatibilities with Audiveris.
  • Integrating native libraries into a cloud-ready Docker environment.
  • Managing file uploads and processing pipelines efficiently.

Results

  • Functional prototype that demonstrates full workflow: image ‚Üí backend ‚Üí MusicXML output.
  • Responsive iOS UI with upload, progress tracking, and export features.
  • Proof of technical feasibility for real-time mobile OMR integration.
